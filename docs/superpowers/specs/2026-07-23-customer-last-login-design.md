# 客户最后成功登录时间设计

**状态：** 已实施  
**日期：** 2026-07-23

## 1. 目标

记录每个客户账号最近一次成功登录系统的时间，并仅在开发者（管理员）的账号管理页面显示。此功能只保留最后一次成功登录时间，不建立完整登录历史，也不记录 IP、失败原因或失败次数。

## 2. 记录规则

- `users` 表增加可为空的 `last_login_at` 时间字段。
- 只有角色为 `CUSTOMER` 的账号成功通过身份验证并创建会话后，才把该字段更新为本次登录时间。
- 密码错误、账号停用、登录限流、参数无效和其他未建立会话的尝试都不能更新。
- 开发者账号登录不更新该字段。
- 已存在客户在迁移后字段为空，不根据旧会话或业务操作时间推算。
- 账号被停用、启用、重置密码或修改资料时保留原最后登录时间。

## 3. 一致性

成功登录中的“创建会话”和“更新最后登录时间”必须在同一个 SQLite `IMMEDIATE` 事务内完成。任一步骤失败时二者一起回滚，避免出现已经记录登录时间但没有有效会话，或已有会话却遗漏时间的情况。

重复登录时使用服务端解析出的当前时间覆盖旧值。系统只显示最新值，不保留历史行。

## 4. 管理员界面

开发者账号的“账号管理”列表增加“最后登录”列：

- 客户已有记录：按中国本地时间显示到分钟。
- 客户没有记录：显示“从未登录”。
- 开发者账号：显示“—”，避免把该列误解为开发者审计日志。

移动端继续使用表格现有的 `data-label` 响应式布局，标签为“最后登录”。不新增独立日志页面。

## 5. 权限与数据暴露

- 继续复用账号管理查询的开发者权限校验。
- `lastLoginAt` 只加入管理员账号列表 DTO。
- 登录会话 DTO、客户页面、需求 DTO、公共接口和账号写操作返回值不增加该字段。
- 客户不能读取自己或其他客户的最后登录时间。

## 6. 测试与验收

- 迁移测试：现有用户迁移后 `last_login_at` 为空，新时间可正常保存和读取。
- 登录测试：客户成功登录后更新；后续成功登录覆盖为较新时间。
- 失败测试：密码错误、停用账号和限流不更新。
- 角色测试：开发者成功登录不更新。
- 原子性测试：会话创建或登录时间更新失败时不留下半完成状态。
- 查询测试：只有开发者可以取得管理员账号列表中的 `lastLoginAt`。
- 组件测试：客户显示格式化时间或“从未登录”，开发者显示“—”。
- 发布前运行类型检查、Lint、Vitest、隔离生产构建和 Playwright。

本功能与“客户重新打开已完成需求”一起发布到 `http://47.121.188.131:13001`。发布前必须创建并验证包含远端 SQLite、所有受引用截图和 manifest 的完整备份。备份失败立即停止，不执行迁移、不替换服务；发布只能更新代码，禁止用本机数据库或截图覆盖公网数据。发布后核对数据库完整性、附件完整性、核心表数量和实际运行版本。
