# 用户手册

## 客户

### 登录和改密

使用开发者分配的用户名和临时密码登录。临时密码首次登录后必须修改；用户名不能自行修改。改密成功后所有会话退出，请用新密码重新登录。

### 提交需求

1. 进入“新建需求”。
2. 选择项目、需求类型和优先级。
3. 填写至少 10 个字符的具体内容。
4. 可在正文框粘贴截图，也可拖放或点击选择。提交前可以预览和移除。
5. 点击“提交需求”。

截图仅支持 PNG、JPEG、WebP。移动设备无法粘贴图片时，使用“选择截图”。

### 查看和处理

需求列表包含所属项目的全部需求及提交人。可以搜索编号或正文，并按项目、类型、优先级、进度和记录状态筛选。点击任一数据表头可以按该列排序，再次点击切换方向；筛选、翻页或刷新不会丢失当前排序。

“待您回复”仍具有最高提醒优先级，正常记录仍排在已暂停记录之前。同一组内默认先显示已排期，再显示未排期和完成；同一进度内按加急、重要、普通排列，最后按更新时间从新到旧排列。

“待您回复”的需求会排在前面并标红。进入详情，在“确认与澄清”填写回复；回复成功后标红立即消失。开发者以后再次提问时才会重新标红。

每行操作位于最右侧。本人提交的需求会显示“编辑”：仅“正常 + 未排期”时可点击，已排期、完成、暂停、归档或项目停用时保留禁用按钮说明原因。本人提交的“正常 + 已排期”需求同时显示“暂停”。暂停后原需求只读，可以直接提交新的需求。

“备注”是开发者给出的说明，客户可以查看但不要求回复。“操作历史”显示创建、状态和公开沟通等记录。

## 开发者

### 初始设置

1. 在“项目管理”创建项目。
2. 在“账号管理”创建客户，设置临时密码。
3. 打开客户的项目权限，将客户分配到相应项目。

账号角色创建后不可修改。停用账号会立即撤销会话。不能停用当前账号或最后一个启用的开发者。

### 处理需求

在需求详情可以：

- 用进度选择器设置未排期、已排期或完成。
- 暂停已排期需求、恢复暂停、归档或恢复归档。
- 在“备注”中添加公开说明。
- 在“确认与澄清”向客户提问。
- 在“私人笔记”维护仅本人可见的内容。
- 查看公开操作历史。

公开备注采用追加方式。私人笔记每位开发者每条需求一份，可以反复保存；其他开发者也看不到。开发者不直接修改客户的原始正文、类型、优先级或截图。

### 澄清原则

只在确实需要客户补充信息时提问。客户回复后提醒自动清除，不需要点击“解决”。若信息仍不足，继续提出新问题即可重新提醒客户。

### 归档查询

归档需求默认不出现在列表。将“记录状态”筛选为“已归档”即可查看和恢复。
