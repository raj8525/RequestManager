# 产品规则

## 产品目标

RequestManager 用一条短路径完成客户和开发者之间的需求协作：提交、查看、处理、澄清。首版不引入审批、处理人、工时、迭代、通知中心或子任务。

## 角色

- 客户：查看所属项目的全部需求，提交需求，按规则编辑、暂停或重新打开自己提交的需求，回复开发者的澄清问题，修改本人密码。
- 开发者：查看全部需求，管理项目和账号并查看客户最后成功登录时间，调整进度与记录状态，添加备注，维护仅本人可见的私人笔记，发起澄清。

账号只有 `CUSTOMER` 和 `DEVELOPER` 两种角色。角色创建后不可修改。开发者同权，不另设管理员角色。

## 需求字段

- 标题：必填，去除首尾空白后 1 到 100 个字符。
- 类型：Bug、功能变更、新增功能。
- 优先级：加急、重要、普通。
- 正文：必填，去除首尾空白后 10 到 10,000 个字符。
- 进度：未排期、已排期、完成。
- 记录状态：正常、已暂停、已归档。
- 截图：PNG、JPEG、WebP；单张最大 10 MiB，每条最多 8 张，总计最大 30 MiB。

新需求固定为“未排期 + 正常”。编号格式为 `REQ-000001`，列表和详情显示独立标题。迁移前的无标题需求显示“待补充标题”。

## 生命周期

- 客户只能完整编辑本人提交且仍为“正常 + 未排期”的需求。可编辑标题、正文、类型、优先级和截图。
- 本人提交的历史无标题需求可在任意进度和记录状态下一次性补充标题；补录时不能修改其他字段。项目停用后仍为只读。
- 客户只能暂停本人提交的“正常 + 已排期”需求。暂停后可以直接提交新需求。
- 客户可将本人提交、所属项目启用的“正常 + 完成”需求重新打开为“正常 + 未排期”。原因必填并可附截图，原完成说明和截图保留。
- 开发者可在正常状态下切换任意进度，可暂停已排期需求，可恢复暂停、归档和恢复归档。
- 暂停只适用于已排期需求；暂停或归档后需求只读。
- 恢复暂停后仍为“已排期 + 正常”。恢复归档后变为“正常”，原进度保留。
- 项目停用后不能接收新需求，已有需求仍可查但客户只读。
- 用户、项目和需求不物理删除。

所有可变更操作使用版本号防止旧页面覆盖新状态，并记录公开操作历史。

## 简单澄清

1. 开发者提问后，正常需求标记为待客户回复。
2. 客户完整结果集中，该需求置顶、标红，并同时显示“待您回复”。
3. 项目中的任一客户回复后，提醒立即清除。
4. 开发者不继续追问时，需求保持正常显示；再次提问才重新标红。
5. 暂停或归档时提醒被抑制；恢复后若最后一条消息仍是开发者问题，则重新显示。

公开备注与澄清不同：公开备注让客户看见，但不会触发待回复。私人笔记只有写入它的开发者本人可见。

客户重新打开原因作为特殊澄清消息显示，标签为“客户重新打开”，不会触发待客户回复。

## 开发者提问

开发者可针对启用项目创建“开发者提问”，正文和每次回复均可附带截图。新提问及开发者追问在项目客户列表中着色置顶；任一项目客户回复后，客户侧取消置顶并在开发者侧着色置顶。开发者手动标记已查看后恢复普通显示。客户补充回复或开发者再次追问会重新激活对应角色的提醒。停用项目中的提问只读。

## 列表与搜索

客户待回复记录继续优先显示，正常记录仍排在已暂停记录之前。同一提醒和记录状态组内默认依次按进度、优先级和更新时间排序：已排期、未排期、完成；同进度内为加急、重要、普通；同优先级内最新更新优先，最后按 ID 倒序保证分页稳定。开发者使用进度、优先级、更新时间和 ID 的默认业务顺序。

编号与需求、项目、提交人、类型、优先级、进度、记录状态和更新时间表头均可点击切换排序方向。排序在完整结果集分页前执行，并通过 URL 在筛选、翻页和刷新后保留。双方都可搜索编号、标题和正文，并按项目、类型、优先级、进度、记录状态筛选。默认隐藏归档，每页 25 条。

列表操作统一位于最右侧。本人提交的需求始终显示编辑入口；只有“正常 + 未排期”时可以点击，其他状态显示禁用按钮并说明限制。
