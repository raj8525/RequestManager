# 安全说明

## 身份与会话

- 密码统一限制 10 到 128 个字符，使用 Node.js scrypt、16 字节随机盐和 64 字节派生密钥。
- 哈希验证使用常量时间比较；数据库和日志不保存明文密码。
- 会话令牌至少 32 个随机字节，Cookie 为 `HttpOnly`、`SameSite=Lax`，数据库只保存 SHA-256 摘要。
- 会话默认七天。停用账号、密码重置和本人改密会撤销全部旧会话。
- 本人改密使用旧密码哈希和实时会话的 CAS 复查，管理员并发重置不会被旧请求覆盖。
- 登录失败统一返回“用户名或密码错误”。用户名和密码先做长度/格式限制，再执行 scrypt。
- 限速同时按用户名加来源和来源总预算执行，15 分钟窗口，过期记录清理，表最多 10,000 行。

## 请求与授权

所有写操作检查登录态、实时账号、强制改密、角色、项目成员关系、资源状态、版本和封闭枚举。Server Actions 校验同源 `Origin`；不信任客户端角色、项目 ID 或隐藏按钮。

无权访问需求或附件时按不存在处理，降低 IDOR 枚举风险。账号停用或解除项目分配后，已有会话下一次访问立即失效或失去资源权限。

## 私人笔记

私人笔记查询和写入都使用当前会话开发者 ID，不接受浏览器指定所有者。详情页只为开发者加载本人笔记。其他开发者和客户的 HTML、RSC 和网络载荷中不包含该正文。操作历史不会记录私人笔记内容。

## 截图

- 请求体、单文件、数量和总大小均有限制。
- 服务端检查文件魔数，不信任扩展名或浏览器 MIME。
- 存储名随机生成，原始文件名不参与路径。
- 文件位于 `public` 外，只能通过每次鉴权的 Route Handler 读取。
- 路径工具拒绝危险根路径、祖先符号链接、canonical alias 和数据库/上传/备份重叠。

## 运维安全

备份包含精确文件集合、大小、SHA-256 和迁移 journal。恢复要求显式双确认、停服、进程互斥锁和完整预检。暂存、交换、回滚和发布均执行必要的文件/目录 fsync。

进程锁不能约束直接打开 SQLite 的第三方工具。恢复和手工数据库操作前仍必须停止应用，并确认没有其他写入者。

结构化日志会过滤密码、令牌、Cookie、正文、私人笔记和附件字节。用户文本作为纯文本渲染，不执行 HTML 或 Markdown。

## 部署清单

- `APP_ORIGIN` 必须是浏览器实际访问的 HTTPS Origin。
- 生产环境自动使用 Secure Cookie。
- 仅在可信反向代理覆盖转发头时启用 `TRUST_PROXY_HEADERS`。
- 数据目录权限限制为运行账户可读写。
- 不公开数据库、备份、上传或进程锁文件。
- 定期运行 `npm audit`、备份和恢复演练。
