# 权限矩阵

授权以服务端领域服务为准。界面隐藏按钮只是辅助，不能替代权限检查。

| 能力 | 客户 | 开发者 |
| --- | --- | --- |
| 登录、退出、修改本人密码 | 可以 | 可以 |
| 修改本人用户名或角色 | 不可以 | 不可以直接修改角色 |
| 创建账号、改用户名、重置密码、启停账号 | 不可以 | 可以 |
| 创建、编辑、停用项目 | 不可以 | 可以 |
| 分配客户到项目 | 不可以 | 可以 |
| 查看需求 | 所属项目全部需求 | 全部需求 |
| 新建需求 | 所属且启用的项目 | 不提供 |
| 编辑需求主体和截图 | 仅本人提交且正常、未排期 | 不修改客户原始主体 |
| 修改进度 | 不可以 | 正常需求可以 |
| 暂停 | 本人提交且正常、已排期 | 任意正常、已排期需求 |
| 恢复暂停、归档、恢复归档 | 不可以 | 可以 |
| 查看公开备注和公开历史 | 有需求查看权时可以 | 可以 |
| 添加公开备注 | 不可以 | 正常需求可以 |
| 查看、编辑私人笔记 | 不可以 | 仅本人笔记，正常需求可以编辑 |
| 发起澄清 | 不可以 | 正常需求可以 |
| 回复澄清 | 所属项目任一客户，且当前待回复 | 通过再次提问推进 |
| 读取截图 URL | 当前仍有项目权限 | 启用开发者可以 |
| 查看开发者提问 | 所属项目 | 全部项目 |
| 创建、继续追问 | 不可以 | 启用项目可以 |
| 回复开发者提问 | 所属且启用项目 | 通过继续追问推进 |
| 标记提问已查看 | 不可以 | 待开发者查看时可以 |

## 实时保护

- 每次页面、Server Action、Route Handler 和附件请求都重新检查账号是否启用、角色是否匹配、是否仍需强制改密。
- 客户解除项目分配后，现有会话下一次访问立即失去项目、需求和截图权限。
- 项目停用后，客户保留历史读取，但不能新建、编辑、暂停或回复。
- 不能停用当前登录的开发者，也不能停用最后一个启用的开发者。
- 开发者重置密码、本人改密或账号停用会撤销该用户全部会话。
- 猜测需求编号、附件 ID 或篡改表单不能扩大权限；无权资源按不存在处理。

## 私人数据边界

私人笔记查询必须同时限定当前开发者 ID。其他开发者、客户、页面 HTML 和 RSC/网络载荷都不会收到该正文。通用操作历史也不记录私人笔记正文。
